本任务旨在为 `RUN` 模式设计并实现一个简洁的状态仪表盘，用于在墨水屏上快照显示系统的核心状态。

-   **任务清单:**
    1.  在 `ui_manager.cpp` 中创建一个新函数 `ui_manager_show_run_dashboard()`。
    2.  UI 需显示至少以下信息：
        -   当前土壤湿度 (%)
        -   设定的浇水阈值 (%)
        -   当前电池电压 (V)
        -   上次浇水时间（可暂时显示为 "N/A"）
        -   系统状态（如 "Sleeping..."）
    3.  在 `run_mode_manager.cpp` 中，实现一套**智能更新机制**来驱动仪表盘的显示。
    4.  **策略要求（混合刷新）：**
        -   **首次进入** `RUN` 模式时，执行一次**全局刷新**以完整显示仪表盘界面。
        -   **后续更新**应遵循**“按需、局部刷新”**的原则：`RunModeManager` 需要缓存上次显示的数值，仅在某个数值（如湿度、电压）发生**显著变化**后，才触发一次**局部刷新**。
        -   **定期清理残影：** 为避免多次局部刷新后产生视觉残影，`RunModeManager` 还需实现一个**定期全局刷新**的机制（例如：每执行10次局部刷新后，或每隔30分钟，进行一次全局刷新）。
        -   此混合策略旨在完美平衡刷新速度、视觉质量和功耗。
    5.  **注意：** 为简化演示逻辑，`RUN` 模式下的手动刷新（单击编码器按键）功能可以**暂时不实现**。

-   **验收标准:**
    1.  首次进入 `RUN` 模式，屏幕会进行一次完整的刷新。
    2.  在 `RUN` 模式下，只有当传感器读数发生变化时，屏幕上的对应数值区域才会进行局部刷新。
    3.  在连续多次（例如10次）局部刷新后，系统应能自动执行一次全局刷新来清理屏幕。
    4.  执行 `force water` 测试命令并导致状态变化后，屏幕应能触发一次相应的局部刷新。
